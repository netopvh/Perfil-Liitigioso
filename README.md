## Description

API protótipo para perfil litigioso de empresas. Dados simulados e determinísticos por CNPJ; persistência em MySQL e cache em Redis (1h).

## Pré-requisitos

- Node.js 20+
- Docker e Docker Compose

## Como rodar

### Opção 1: Tudo no Docker (recomendado)

MySQL, Redis e a API rodam em containers em uma **rede interna** (`perfil-internal`). Apenas a porta da API fica exposta (3000).

```bash
cp .env.example .env   # opcional: PORT, senhas
docker compose up -d --build
```

API: `http://localhost:3000`. Swagger: **http://localhost:3000/api**. No `.env`, `PORT` altera a porta exposta.

### Opção 2: API local + MySQL e Redis no Docker

1. Copiar o override para expor as portas do MySQL e Redis:  
   `cp docker-compose.override.example.yml docker-compose.override.yml`
2. Subir só a infraestrutura:  
   `docker compose up -d mysql redis`
3. Copiar `.env.example` para `.env` e definir `DB_HOST=localhost`, `REDIS_HOST=localhost`.
4. Rodar a API:  
   `npm install && npm run start:dev`

API: `http://localhost:3000`. Swagger: **http://localhost:3000/api**.

## Endpoints (prefixo `/v1`)

- `GET /v1/companies?query=` — Buscar empresas
- `GET /v1/companies/:cnpj` — Empresa por CNPJ
- `GET /v1/companies/:cnpj/litigation-profile` — Perfil litigioso
- `POST /v1/companies/:cnpj/simulate/recompute` — Simular recálculo (body: `{ "scenario": "aggressive_settlement" }`)
- `GET /v1/companies/:cnpj/cases` — Listar processos (filtros: ramo, status, tribunal, from, to, page, limit)
- `GET /v1/cases/:processId` — Detalhe do processo

